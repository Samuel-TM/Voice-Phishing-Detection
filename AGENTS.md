# AGENTS.md

本文件用于指导后续 AI/Codex 在本项目中工作。修改代码前请先阅读本文件，尤其是运行环境、模型文件、风险分析主流程和命名迁移约束。

## 项目定位

本项目是毕业设计原型演示系统，核心目标是展示“语音诈骗动态风险追踪”能力，而不是一开始就追求完整生产级部署。

优先保证：

- 系统可以稳定启动和演示。
- 上传音频后可以生成按时间变化的风险曲线。
- 前端展示逻辑与论文中的 dynamic risk tracking / in-event risk evolution 目标一致。
- 模型推理链路能跑通，并能给出可解释的文本风险、音频风险、融合风险和平滑风险。

后续如果时间允许，可以扩展为浏览器麦克风实时输入，甚至进一步迁移到手机端可部署应用。但当前阶段的主要交付形态是“上传完整音频，使用滑动窗口模拟实时流式分析”。

## 当前核心流程

当前系统主流程是“上传音频后模拟实时流式分析”：

1. 用户上传一段完整音频。
2. 后端按滑动窗口切分音频。
3. 每个窗口执行 STT。
4. 文本风险基于截至当前窗口的累计转写文本计算，体现语义风险累积。
5. 音频风险基于当前窗口计算，体现局部声学异常或深伪风险。
6. 融合文本风险和音频风险。
7. 使用平滑策略生成更适合展示的风险演变曲线。
8. 前端按窗口顺序播放 timeline，模拟实时风险追踪。

旧接口可以保留：

- `/predict`：单条文本检测。
- `/api/audio_result`：整段音频分析。
- `/api/stream_audio_analysis`：当前原型系统的主流程接口。

如果后续需要明确答辩主线，应优先围绕 `/api/stream_audio_analysis` 展示。

## 实时性定义

当前阶段中，论文和系统里的 `real-time` / `dynamic risk tracking` 可以解释为：

> 使用上传音频和滑动窗口进行 simulated streaming / incremental analysis。

也就是说，第一版不要求浏览器麦克风实时输入。后续如有时间，可以新增浏览器麦克风流式输入，但不要破坏现有上传音频模拟实时流的演示路径。

## 命名迁移策略

项目里存在历史命名，例如：

- `KoBERTModel`
- `deepvoice_detection`

这些名字来自原始开源项目或旧实现。后续可以逐步重构为更准确的命名，例如中文 BERT 文本风险模块、audio risk 模块等。

但重命名时必须同步修改所有关联代码、导入路径、模板调用、测试脚本和文档引用。不要只改目录名或文件名。若时间紧，优先保证系统可运行，再做命名清理。

推荐策略：

- 先新增兼容层或清晰包装函数。
- 再逐步迁移调用方。
- 最后删除旧命名。

## 模型文件约束

模型文件是本地必备运行资产，不应纳入 Git，也不应由后续 AI 自动重新训练或下载替代。

当前关键模型文件：

- 文本模型权重：`KoBERTModel/model/train.pt`
- 音频模型权重：`deepvoice_detection/model/best_f1_model.pt`
- 音频模型配置：`deepvoice_detection/model/deepvoice_config.json`

如果模型文件缺失，应该返回清晰、结构化的错误信息，并提示缺少哪个文件。不要自动启动训练流程，不要静默下载其他模型来替代项目指定权重。

例外：基础依赖模型缓存（例如 Hugging Face tokenizer/model 或 Whisper 模型）可以在环境准备阶段处理，但业务训练权重不应被替代。

## 风险分数和阈值

当前动态融合和平滑策略是原型默认设置，不是不可修改的最终规则。

当前默认值：

- 融合分数：`0.8 * text_score + 0.2 * voice_score`
- 平滑分数：`0.65 * previous + 0.35 * current`
- 风险等级：
  - `Normal < 50`
  - `Suspicious 50-69`
  - `High Risk 70-89`
  - `Critical >= 90`

后续 AI 可以根据实验结果、论文表述和演示效果调整权重、平滑参数和阈值。但调整时应同步更新：

- 后端评分逻辑。
- 前端显示文案。
- README 或论文相关说明。
- 测试用例或验证说明。

## 异常窗口处理

滑动窗口分析中，某些窗口可能出现：

- 静音。
- 音频过短。
- STT 无结果。
- 局部推理失败。

这些情况不应中断整段音频的分析。推荐做法是：

- 在对应 timeline 点标记异常状态。
- 保留窗口的 `start_sec` 和 `end_sec`。
- 尽可能返回可用的其他通道分数。
- 如果文本为空，文本风险可以按低风险或不可用处理。
- 前端应能继续播放后续窗口。

不要因为单个窗口失败而让 `/api/stream_audio_analysis` 整体返回 500，除非输入文件本身无法读取或核心模型完全不可用。

## 常见风险点

- 不要在未激活 `dissertation` 环境时运行脚本。
- 不要假设模型文件在 Git 中存在。
- 不要把中文 STT 内容翻译成英文。
- 不要为了重命名旧目录而漏改导入路径。
- 不要让单个静音窗口或 STT 失败中断整段音频分析。
- 不要把旧的整段音频一次性分析误认为当前主研究贡献。
- 不要将演示系统过早改成只能依赖真实麦克风输入，否则会增加答辩现场风险。
